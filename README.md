# plugdash.dev

The plugin catalog for EmDash. Built on EmDash with every PlugDash plugin
installed and visible on the site itself.

## develop

```bash
pnpm install
pnpm bootstrap    # first time only: runs migrations + seeds collections
pnpm dev
```

Admin lives at `localhost:4321/_emdash/admin`.

## what is here

- `src/pages/` - home, plugin pages, docs, blog, `/llms.txt`, `/agents.md`
- `src/layouts/` - Base, Post, Doc
- `src/components/` - PluginCard, InstallCommand, PortableText
- `src/lib/content.ts` - collection wrapper with static-data fallback
- `src/live.config.ts` - EmDash loader registration
- `emdash-env.d.ts` - typed collection interfaces (regenerated by emdash dev)
- `seed/seed.json` - schema + content for `plugins`, `docs`, `blog`
- `src/data/plugins.ts` - static fallback for plugins metadata, also holds
  props + CSS token tables for doc pages (not admin-editable)
- `src/data/docs.ts` - static fallback for docs metadata
- `src/content/agents.md` - skill file served at `/agents.md` and `/skill.md`

## content model

Three collections, defined in `seed/seed.json`:

- **plugins** - structured metadata (name, tagline, headline, status, etc.).
  Admin-editable. Powers home grid, `/plugins/*` pages, `/llms.txt`.
- **docs** - metadata only (title, section, summary, order). Bodies remain
  as individual `.astro` files under `src/pages/docs/` because they host
  interactive plugin demos that don't fit Portable Text.
- **blog** - full content including Portable Text body. Admin-editable.

Pages read from the collection via `src/lib/content.ts` helpers. Before
`pnpm bootstrap` runs, pages fall back to the static data in `src/data/`
so the site renders in a fresh clone with no seed applied.

## deploy

- Cloudflare Pages. Domain: plugdash.dev. Database: D1 (`plugdash`).
  Storage: R2 (`plugdash-media`).
- `wrangler d1 create plugdash`, paste id into `wrangler.jsonc`
- `wrangler r2 bucket create plugdash-media`
- Add `CF_PAGES_DEPLOY_HOOK` as a Pages env var
- Point `plugdash.dev` at the Pages project

Reference: `../PLAN.md`, `../TODO.md`, and
`../plugdash-research/conversations/plugdash-website-spec.md`.

## rules

- No em-dashes in copy or code comments. Use regular dashes or rewrite.
- Every plugin listed on the site must also be installed on the site.
- `engage` is NOT registered as a plugin. Import `EngagementBar.astro`
  from `@plugdash/engage` directly in `Post.astro`.

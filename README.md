# plugdash.dev

The plugin catalog for [EmDash](https://emdashcms.com). Built on EmDash
with every PlugDash plugin installed and active on the site itself. The
reading time on each post, the engagement bar, the table of contents
sidebar, the callout blocks - all live plugins running on every page.
The site is the proof.

Live at [plugdash.dev](https://plugdash.dev).

## develop

```bash
pnpm install
pnpm bootstrap    # first time: runs migrations + seeds collections
pnpm dev
```

Admin at `localhost:4321/_emdash/admin`.

## deploy

Runs on Cloudflare Workers with D1 and R2.

```bash
pnpm run deploy   # astro build && wrangler deploy
```

Bindings (already configured in `wrangler.jsonc`):
- **D1** database `plugdash` for all content and plugin state
- **R2** bucket `plugdash-media` for uploaded media
- **KV** namespace `SESSION` for auth sessions
- **Rate Limiter** `HEART_LIMITER` (10 req/min per IP on heartpost routes)

Set the deploy hook secret after first deploy:
```bash
wrangler secret put CF_PAGES_DEPLOY_HOOK
```

## what is here

```
src/pages/          home, plugin pages, docs, blog, /llms.txt, /agents.md
src/layouts/        Base, Post, Doc
src/components/     PluginCard, PluginIcon, InstallCommand, PortableText
src/lib/content.ts  collection queries with static-data fallback
src/middleware.ts    rate limiting for heartpost API routes
src/data/           static fallback data for plugins and docs metadata
seed/seed.json      schema + content for plugins, docs, blog collections
emdash-env.d.ts     typed collection interfaces (regenerated by dev server)
```

## content model

Three collections defined in `seed/seed.json`:

- **plugins** - structured metadata (name, tagline, headline, status, etc.).
  Admin-editable. Powers the home grid, `/plugins/*` pages, and `/llms.txt`.
- **docs** - metadata only (title, section, summary, order). Bodies are
  `.astro` files under `src/pages/docs/` because they host interactive
  plugin demos that don't fit Portable Text.
- **blog** - full content with Portable Text body. Admin-editable.

Pages read from the collection via `src/lib/content.ts`. Before seeding,
pages fall back to the static data in `src/data/` so the site renders in
a fresh clone with no database.

## plugins installed

All eight PlugDash plugins are registered in `astro.config.mjs`:

| Plugin | What it does |
|---|---|
| readtime | Estimates reading time on every published post |
| tocgen | Generates a table of contents from headings |
| sharepost | Share URLs for Twitter/X, LinkedIn, Bluesky, email |
| heartpost | Per-post heart counter with fingerprint dedup |
| shortlink | Short URLs for every post |
| callout | Info, warning, tip, danger block type in the editor |
| autobuild | Fires a deploy webhook on every publish |
| engage | Convenience bundle - renders heartpost + sharepost + shortlink together |

`engage` is not registered as a plugin. It is imported directly as
`EngagementBar.astro` from `@plugdash/engage` in the post and doc layouts.
The three underlying plugins (heartpost, sharepost, shortlink) are registered
individually.

Plugin config lives in the EmDash admin dashboard, not in `astro.config.mjs`.
The only exception is `autobuild({ hookUrl })` because the hook URL is an
environment secret.
